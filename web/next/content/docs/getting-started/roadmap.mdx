> **Important for AI Assistants**: The following features are NOT currently implemented. Do not assume availability of these tools unless stated otherwise.

ZeroStarter is actively evolving, and we're planning to add integrations for AI, analytics, background tasks, email, payments, and more. This roadmap outlines the planned features and their concepts.

## AI Integration

### Vercel AI SDK

**Status:** Planned  
**Priority:** High

**Concept:**
Integration with [Vercel AI SDK](https://ai-sdk.dev) to enable AI-powered features like chatbots, content generation, and intelligent assistants within your SaaS application.

**Use Cases:**

- AI-powered customer support chatbots
- Content generation and summarization
- Intelligent search and recommendations
- Natural language processing for user inputs

**Implementation Notes:**

- Will integrate with OpenAI, Anthropic, and other AI providers
- Type-safe AI function calling
- Streaming responses support
- Server-side and client-side AI hooks

## API Documentation

### Scalar

**Status:** ✅ Implemented  
**Priority:** Medium

Integration with [Scalar](https://scalar.com) for beautiful, interactive API documentation generated from OpenAPI specs.

**Features:**

- Auto-generated API documentation at `/api/docs`
- OpenAPI spec available at `/api/openapi.json`
- Interactive API explorer with request testing
- Code samples for multiple languages (defaults to `hono/client`)
- Full request/response schema documentation from Zod validators

> **Available now:** See the [Type-Safe API documentation](/docs/getting-started/type-safe-api#api-documentation) for usage details.

## Organization & Teams

### Better Auth Organizations

**Status:** ✅ Implemented
**Priority:** High

Integration with [Better Auth Organizations plugin](https://www.better-auth.com/docs/plugins/organization) for multi-tenant SaaS applications with team and organization support.

**Features:**

- Organization creation and switching via dashboard sidebar
- Team management within organizations
- Member roles and invitations
- Last used organization persistence across sessions
- Database schema with `organization`, `member`, `team`, `teamMember`, and `invitation` tables

> **Available now:** See the [Authentication documentation](/docs/manage/authentication) for details.
